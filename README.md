# Hadoop-Based Batch Analytics Pipeline for CDR

A production-style batch analytics pipeline for telecommunications Call Detail Records (CDRs) using Hadoop, Apache Spark, and Apache Airflow.

## Architecture
- **Storage**: HDFS (Hadoop 3.2.1)
- **Processing Engine**: Apache Spark 3.5.0
- **Orchestration**: Apache Airflow 2.8.0
- **Environment**: Dockerized (Multi-container)

## Key Features
- **Data Generation**: Generates 2,000,000 CDR records with intentional data skew (a "Whale Caller" account for 10% of records).
- **Optimization**: 
  - **Salting**: Mitigates data skew in `top_callers` job.
  - **Custom Partitioner**: Implemented in `anomalous_calls` to ensure key-locality for statistical aggregations.
  - **Native CSV Parsing**: Optimized Spark job efficiency.
- **Observability**: Every job produces a `_MANIFEST.json` with execution metadata (input/output counts, timestamps, status).

## Directory Structure
- `data/`: Raw CDR data (generated by container).
- `jobs/`: PySpark scripts for the 4 logical queries.
- `dags/`: Airflow DAG definitions.
- `output/`: Local volume for viewing results (HDFS-integrated).

## Setup & Execution
1. **Start the Cluster**:
   ```bash
   docker-compose up -d --build
   ```
2. **Run a Query**:
   ```bash
   ./run_pipeline.sh <query_name>
   # query_name: top_callers, tower_heatmap, anomalous_calls, revenue_recon
   ```
3. **Access UIs**:
   - Airflow: [http://localhost:8080](http://localhost:8080) (Admin/Admin)
   - Spark Master: [http://localhost:8081](http://localhost:8081)
   - Hadoop Namenode: [http://localhost:9870](http://localhost:9870)

## Design Decisions
- **HDFS Integration**: Jobs write to HDFS to handle distributed write permissions and large datasets effectively.
- **SequentialExecutor**: Used for Airflow to simplify local setup with SQLite.
- **Custom RDD Partitioner**: Used in `anomalous_calls.py` to satisfy strict requirements for manual partitioning logic.
